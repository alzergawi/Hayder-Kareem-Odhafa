📊 2. Data We Collect & How We Use It

RusLex is designed with privacy in mind. We minimize data collection to only what is necessary for the app to function.

🔗 3. Third-Party Services & SDKs

RusLex integrates the following third-party services. Each may collect data according to their own privacy policies:

a) Google AdMob (Advertising)

We use Google AdMob to display optional rewarded advertisements. AdMob may collect:

  • Advertising ID (device identifier for ad personalization)
  • IP address (approximate location for ad targeting)
  • Device information (model, OS version, screen size)
  • Ad interaction data (impressions, clicks)

AdMob's data practices are governed by Google's Privacy Policy and How Google Uses Data When You Use Our Partners' Apps.

b) Google User Messaging Platform (UMP / GDPR Consent)

We use Google's UMP SDK to collect and manage advertising consent from users in regions that require it (EU/EEA, UK). This ensures compliance with GDPR and ePrivacy regulations before any personalized ads are served.

c) Google ML Kit (Translation)

We use ML Kit's on-device translation API to provide real-time translations. All translation processing happens locally on your device; no text is sent to external servers for translation.

d) Android Speech Recognition

The app uses the device's built-in speech recognition engine for pronunciation practice. Audio is processed by the device's local speech recognizer and is never recorded, stored, or transmitted by RusLex.

🛡️ 5. Data Storage & Security

RusLex stores all user-generated data locally on your device:

  • Language database: Stored in a local SQLite database on your device
  • Learning progress (SRS cards, streaks, reviews): Stored locally; never uploaded to any server
  • App preferences (language, notification settings): Stored in local SharedPreferences

We do not operate any server or cloud storage. The only network communication from RusLex is:

  • Outgoing ad requests to Google AdMob servers
  • Periodic checks for language database updates from a secure endpoint

All network communication is encrypted using HTTPS/TLS.

🗑️ 6. Data Retention & Deletion

Since RusLex does not create accounts or store data on servers, data retention is fully under your control:

  • To delete all app data: Go to Android Settings → Apps → RusLex → Storage → Clear Data
  • To uninstall: Uninstalling the app automatically removes all locally stored data
  • To reset Advertising ID: Go to Android Settings → Privacy → Ads → Reset advertising ID or delete it entirely
  • To opt out of personalized ads: Go to Android Settings → Privacy → Ads → Opt out of Ads Personalization

For data collected by Google AdMob, please refer to Google's data retention policies.

🤖 7. AI-Generated Content Disclaimer

RusLex uses Artificial Intelligence technology to generate translations, linguistic explanations, and educational content displayed within the app. While we strive for accuracy:

  • AI-generated translations and explanations may contain inaccuracies or errors
  • Content is provided "as is" for educational purposes only and should not be considered a substitute for professional language instruction
  • Users are encouraged to verify critical translations with qualified human translators

This disclaimer is also displayed within the app interface.

🔗 3. Сторонние сервисы и SDK

RusLex интегрирует следующие сторонние сервисы. Каждый из них может собирать данные в соответствии со своими политиками конфиденциальности:

а) Google AdMob (Реклама)

Мы используем Google AdMob для показа необязательной рекламы с вознаграждением. AdMob может собирать:

  • Рекламный идентификатор (для персонализации рекламы)
  • IP-адрес (приблизительное местоположение для таргетинга)
  • Информация об устройстве (модель, версия ОС, размер экрана)
  • Данные взаимодействия с рекламой (показы, клики)

Практики AdMob регулируются Политикой конфиденциальности Google.

б) Google UMP (Управление согласием GDPR)

Мы используем Google UMP SDK для сбора и управления рекламным согласием в регионах, требующих этого (ЕС/ЕЭЗ, Великобритания).

в) Google ML Kit (Перевод)

Мы используем API локального перевода ML Kit. Вся обработка переводов происходит локально на вашем устройстве; никакие тексты не отправляются на внешние серверы.

г) Распознавание речи Android

Приложение использует встроенный движок распознавания речи устройства. Аудио обрабатывается локально и никогда не записывается, не сохраняется и не передаётся RusLex.

🛡️ 5. Хранение и безопасность данных

RusLex хранит все пользовательские данные локально на вашем устройстве:

  • Языковая база данных: хранится в локальной базе SQLite
  • Прогресс обучения (карточки SRS, серии, обзоры): хранится локально; никогда не загружается на серверы
  • Настройки приложения (язык, уведомления): хранятся в локальных SharedPreferences

Мы не имеем и не используем серверы или облачное хранилище. Единственная сетевая связь от RusLex:

  • Исходящие рекламные запросы на серверы Google AdMob
  • Периодические проверки обновлений языковой базы из защищённой конечной точки

Все сетевые соединения зашифрованы с использованием HTTPS/TLS.

🗑️ 6. Хранение и удаление данных

Поскольку RusLex не создаёт аккаунтов и не хранит данные на серверах, контроль над данными полностью в ваших руках:

  • Удаление всех данных: Настройки Android → Приложения → RusLex → Хранилище → Очистить данные
  • Удаление приложения: автоматически удаляет все локальные данные
  • Сброс рекламного ID: Настройки Android → Конфиденциальность → Реклама → Сбросить рекламный идентификатор
  • Отключение персонализации: Настройки Android → Конфиденциальность → Реклама → Отключить персонализацию рекламы

Для данных, собранных Google AdMob, обратитесь к политикам хранения данных Google.
